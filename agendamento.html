Sistema de Agendamento Inteligente

Workflow avançado que gerencia múltiplos profissionais em tempo real, previne conflitos de horário e sincroniza automaticamente com GoHighLevel.

1. Identificação e Contexto
O sistema recebe o pedido de agendamento e imediatamente consulta o Supabase e o GoHighLevel para buscar o histórico completo do cliente. Identifica se já existe uma oportunidade ativa no pipeline e recupera dados como nome, email e telefone formatado.
Supabase GoHighLevel API
2. Mapeamento de Profissional
Com base no nome do médico solicitado (Dr. Felipe ou Dra. Gabriela), o sistema mapeia automaticamente o Calendar ID correto e a especialidade. Aceita variações no input (maiúsculas, com/sem ponto, apenas primeiro nome) e normaliza para garantir precisão.
String Normalization Calendar Mapping
3. Decisor de Fluxo
O Switch verifica o tipo de evento (agendamento, cancelamento ou reagendamento) e direciona para o fluxo correto. Cada tipo tem validações e ações específicas para garantir integridade dos dados.
Conditional Logic Flow Control
4. Verificação de Conflitos
Antes de criar qualquer agendamento, o sistema busca na API de Events do GoHighLevel se o cliente já possui um horário marcado. Usa três estratégias de busca (Event ID, Contact ID, Email) para garantir que nenhum agendamento duplicado seja criado.
Event Lookup Deduplication
5. Validação de Disponibilidade
Consulta a rota /free-slots do calendário do profissional selecionado, retornando apenas os horários realmente livres. Valida se o horário solicitado está na lista de slots disponíveis antes de prosseguir.
GoHighLevel Calendar API Slot Validation
6. Criação ou Atualização
Novo Agendamento: Cria o evento via API com status "confirmed" e atualiza o pipeline para "Consulta Marcada".
Reagendamento: Cancela o evento antigo (se não cancelado pelo GHL), cria novo agendamento e atualiza o Event_ID no Supabase.
Cancelamento: Marca o evento como "cancelled" e atualiza o custom field "status_da_consulta" no CRM.
API Orchestration State Management
7. Sincronização Multi-Sistema
Após confirmar o agendamento, o sistema atualiza três pontos de controle: armazena o Event_ID no Supabase (para referência futura), move a oportunidade para a etapa correta do pipeline no GoHighLevel, e atualiza o custom field "status_da_consulta" para rastreamento visual no CRM.
Multi-System Sync Data Consistency
